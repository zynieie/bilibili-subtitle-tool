# Bilibili Subtitle Extractor

> Lightning-fast extractor for Bilibili AI-generated subtitles. Pulls multi-language subtitles (zh / en / ja / es / ar / pt) in seconds via headless browser automation.
>
> 极速 B 站 AI 字幕提取工具 —— 一键拿下多语言字幕。

## Why this tool

Bilibili does not expose its AI subtitle endpoint through any documented public API, and `subtitle_url` is hidden for unauthenticated users. This tool drives a real browser session with your logged-in cookies, triggers the player to emit AI subtitles, intercepts the network request, and writes the result as plain text.

No API keys, no reverse-engineered endpoints — just your own browser session doing what you would do manually.

## Features

- **Multi-language capture** — One run can fetch all six AI subtitle tracks (zh / en / ja / es / ar / pt) for a single video.
- **Sub-20-second runtime** — v7.1 pulls all three languages (940 lines) in ~17 seconds on a 12-minute video.
- **Three independent backends** — Native API, Playwright, and Selenium + undetected-chromedriver, with automatic fallback.
- **Cookie injection** — Bring your own SESSDATA (saved locally, never transmitted).
- **GUI and CLI** — Tk-based desktop window for interactive use; importable functions for scripting.
- **Persistent Chrome profile** — Log in once, the tool reuses your session on every subsequent run.

## Requirements

- Python 3.8 or newer
- Google Chrome (any recent version) installed locally
- A Bilibili account (free signup) — only needed for AI subtitles
- Windows is the primary tested platform; macOS / Linux should work with minor path adjustments.

Python packages (installed automatically via `pip install -r requirements.txt`):

```
requests
playwright
selenium
undetected-chromedriver
```

After installing Playwright, fetch its bundled Chromium once:

```bash
playwright install chromium
```

## Quick Start

```bash
# 1. Clone
git clone https://github.com/zynieie/bilibili-subtitle-tool.git
cd bilibili-subtitle-tool

# 2. Install Python dependencies
pip install -r requirements.txt
playwright install chromium

# 3. Provide your Bilibili cookies
#    - Open bilibili.com in Chrome while logged in
#    - F12 -> Application -> Cookies -> copy SESSDATA
#    - Save as bili_cookies.json (see bili_cookies.example.json)
cp bili_cookies.example.json bili_cookies.json
#    ... then edit bili_cookies.json and replace the placeholder values

# 4. Run
python main.py
```

The window will open. Paste any Bilibili video URL (or bare `BV...` id), pick a backend, click **Fetch Subtitles**.

A minimal CLI / scripting example lives in [`examples/example_usage.py`](examples/example_usage.py).

## Configuration

The tool reads three optional environment variables to override default Chrome paths:

| Variable | Default (Windows) |
|---|---|
| `BILI_CHROME_EXE` | `C:\Program Files\Google\Chrome\Application\chrome.exe` |
| `BILI_CHROMEDRIVER_EXE` | sibling of `chrome.exe` |
| `BILI_CHROME_USER_DATA` | `%LOCALAPPDATA%\Google\Chrome\User Data` |

Cookies are read from `bili_cookies.json` in the project root. Subtitle output is written to `subtitles/` (auto-created). Both directories are in `.gitignore`.

## Performance

| Version | Task | Time |
|---|---|---|
| v5 | Single language, first-time generation | ~30 s |
| v6 | Single language, cache hit | 12.7 s |
| **v7.1** | **Three languages (zh + en + ja), cache hit** | **17.0 s** |

Cache hit means the AI subtitles have been generated by your account before. Once any track is generated, subsequent runs are roughly 10x faster.

## How it works (short version)

1. Launch a controlled Chrome instance (undetected-chromedriver or Playwright).
2. Inject `SESSDATA` so Bilibili treats the session as logged in.
3. Open the target video, call `video.play()`, hover the player.
4. Use JS to click the subtitle button and pick the language you want.
5. Poll the browser's network log for the `bfs/ai_subtitle/prod` request URL.
6. Download the JSON with `requests`, extract `body[]`, write a `.txt` file.

Detailed rationale for each step is in [`docs/PITFALLS.md`](docs/PITFALLS.md).
